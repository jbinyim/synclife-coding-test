# AI 도구 활용 (AI_USAGE)

> AI 사용은 **권장**됩니다. 이 문서는 "얼마나 썼는지"가 아니라
> **AI가 준 답을 어떻게 검증·판단했는지**를 봅니다. 솔직하게 적어 주세요.

## 운영 방식

기획/구현/수정 작업 단위가 끝날 때마다 아래 표와 사례 목록을 **즉시** 갱신한다. 몰아서 쓰지 않는다. (규칙: `CLAUDE.md` 문서 규칙 참고)

## 1. 사용한 도구

- Claude Code — 요구사항 분석, 설계 논의, 코드 작성/리뷰 전반

## 2. 주로 어디에 썼나

| 작업 | 사용한 도구 | 활용 방식 |
|------|-------------|-----------|
| 요구사항 분석·전체 작업 계획 수립 | Claude Code | README와 스타터 코드 전체를 분석시켜 P1 중심 작업 순서·핵심 설계 방향(태스크별 직렬 요청 큐, 스냅샷 롤백, 컬럼별 가상화) 초안을 받고 논의 후 확정 |
| 작업 규칙 수립 (CLAUDE.md) | Claude Code | 과제 요구사항 체크리스트와 기존에 쓰던 개발 규칙을 결합한 규칙 문서 초안 작성 |

## 3. AI 답을 그대로 쓰지 않고 검증·거부·수정한 사례 (중요)

- (구현 진행하며 사례 발생 시마다 기록: 어떤 답이 틀렸거나 부족했고, 어떻게 알아챘으며, 무엇으로 고쳤는지)

## 4. 스스로 판단해서 결정한 부분

- **Git 전략**: AI는 솔로 작업 + 시간 제약을 이유로 main 직접 커밋을 추천했지만, 평소 워크플로우대로 **기능별 브랜치 → main 머지** 방식을 택했다. 브랜치 단위로 작업 범위가 명확해지고, 히스토리에서 기능별 진행 과정을 보여줄 수 있다고 판단했다.
